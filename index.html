Good Enough Software License (GESL)

The Good Enough Software License (GESL) is a unique licensing model that ensures software meets high standards in security and development while maintaining practical flexibility through the concept of "good enough" software.

What Makes Software "Good Enough"?

The GESL is built on the principle that software doesn't need to be perfect to be valuable and usable. Our specialized Committee evaluates software based on practical adequacy rather than theoretical perfection.

Key Principles:

  • Security First: Software must meet essential security best practices
  • Quality Standards: Code should be maintainable and functional
  • Practical Adequacy: Focus on real-world usability over theoretical perfection

The Committee

Security Expert

Evaluates software for compliance with security best practices and ensures basic security standards are met

Development Expert

Reviews code quality, maintainability, and functionality to ensure sustainable development practices

Good Enough Officer (GEO)

Makes final decisions based on practical adequacy, balancing perfection with practicality

Licensing Options

Standard GESL

Requires Committee approval for use, modification, or distribution. Ideal for organizations committed to maintaining high standards while embracing practical solutions.

Dual Licensing - NGESL Option

Organizations can convert to the "Not Good Enough License" (NGESL) through a fee-based model, removing Committee restrictions while maintaining access to the software. This option provides flexibility for organizations with different needs or constraints.

How to Request the License

  1. Fork this repository
  2. Add your organization to organizations.html
  3. Create a Pull Request with the following information:
    • Your organization name
    • Project GitHub page URL
    • Explanation of why your project should be granted the GESL
  4. The Committee will review your request and provide a decision
